# Governance

SponsorFlow OS is currently maintained by Jad Daccache.

## Maintainer Responsibilities

- Review issues and pull requests
- Keep documentation practical and current
- Maintain the roadmap
- Protect the project from low-value complexity
- Encourage reusable contributions

## Decision Criteria

Changes are accepted when they improve:

- sponsorship execution
- reporting clarity
- workflow repeatability
- team handover
- CRM interoperability
- measurable sponsor outcomes

## Non-Goals

This project is not intended to become:

- a bloated enterprise CRM
- a generic sales tool
- a closed sponsorship agency playbook
- a dumping ground for random templates
