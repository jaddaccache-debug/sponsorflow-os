# Contributing to SponsorFlow OS

This project is designed to help small teams manage sponsorship operations with better structure.

## Ways To Contribute

- Improve documentation
- Add templates
- Add CRM examples
- Add data model improvements
- Add dashboard examples
- Add localization
- Report gaps from real sponsorship workflows

## Contribution Standard

A useful contribution should be:

- practical
- documented
- reusable by other teams
- clear enough for non-technical operators
- simple before it becomes complex

## Pull Request Checklist

Before opening a PR:

- [ ] The change solves a real sponsorship workflow problem
- [ ] The documentation is clear
- [ ] The change does not add unnecessary complexity
- [ ] Examples are included where useful
- [ ] Any schema changes are reflected in `docs/DATA_MODEL.md`

## Commit Style

Use clear commit messages:

```text
docs: add sponsor follow-up cadence
schema: add in-kind contribution field
template: add event visibility inventory
```
